## Короткая (Telegram)

🔦 **Yeelight без облака — новый адаптер для ioBroker**

Надоело, что лампы Yeelight думают секунду-другую, пока команда сгоняет в китайское облако и обратно? Написал адаптер прямого LAN-управления: голый TCP до лампы, отклик мгновенный, интернет не нужен вообще.

Что внутри:
— автопоиск ламп + объекты строго по возможностям модели (никаких мёртвых состояний);
— встроенный пульт прямо в админке: лампа на экране светится реальным цветом и яркостью;
— полная поддержка фоновой подсветки (амбилайт у люстр) — вкл/яркость/CT/RGB/HSV;
— инкрементальные кнопки (ярче/теплее/оттенок) — вешаются на выключатели без скриптов.

Ищу тестеров перед подачей в официальный репозиторий — нужны разные модели ламп. Ставится в две команды, инструкция в README (RU/EN):

https://github.com/KOE73/iobroker.yeelight-lan-direct

Баги и хотелки — в issues или сюда. Скрин пульта в репо 😉

## Длинная (4PDA / форум, RU)

**[Новый адаптер] ioBroker.yeelight-lan-direct — прямое локальное управление Yeelight/Mi, без облака**

Всем привет. Устал от связки «нажал — подумало — через облако — сработало» и от того, что при проблемах с интернетом свет превращается в тыкву. Результат — адаптер для ioBroker, который общается с лампами напрямую по TCP/JSON-RPC в локальной сети (тот же протокол, что у официального приложения в LAN-режиме).

**Возможности:**
- SSDP-автопоиск с чтением реальных возможностей каждой лампы — в дереве объектов только то, что модель умеет: у моно-лампочки не будет мёртвых hue/sat, у люстры с амбилайтом будет полный набор bg_* (питание, яркость, CT, RGB, HSV фонового канала);
- встроенный пульт-вкладка в админке: живое свечение реальным цветом/яркостью, элементы управления по возможностям модели, отдельное кольцо фоновой подсветки;
- инкрементальные состояния (BRIGHT_UP/DOWN, CT_UP/DOWN, HUE_CW/CCW…) — прямая привязка к физическим выключателям и энкодерам без скриптов;
- очередь команд с троттлингом — прошивка Yeelight молча теряет команды при заваливании пачками, адаптер это учитывает;
- поиск работает и на хостах с VPN/несколькими сетевыми интерфейсами (режим «все интерфейсы», ручное указание адреса);
- flow-эффекты (start_cf) — парсинг в обе стороны, сцены и таймер сна.

**Что нужно:** LAN Control, включённый в приложении Yeelight; ioBroker с js-controller 5+.

**Установка** (пока адаптер не в официальном репо):
```
iobroker url https://github.com/KOE73/iobroker.yeelight-lan-direct.git
iobroker add yeelight-lan-direct
```

**Зачем пишу сюда:** перед подачей в официальный репозиторий ioBroker хочу обкатку на разных моделях. У меня проверено на ceiling (ceilc, с амбилайтом) и color4. Если у вас mono / strip / bslamp / desklamp / ceiling других серий — буду благодарен за прогон и фидбек: что заработало, что нет, чего не хватает. Issues на GitHub или прямо в теме.
